# Atividade: Missão Bootstrap
## Resolver um problema real

---

## Objetivo da atividade

Fazer com que os alunos usem Bootstrap para pensar, decidir, testar e melhorar uma interface, e não apenas reproduzir modelos prontos.

---

## Proposta

Em duplas, vocês receberão um desafio prático e, a partir daí, criarão uma interface em Bootstrap para resolver um problema real de organização visual e usabilidade.

A ideia é que vocês planejem, construam, testem com outra dupla e melhorem a solução.

---

## Competências trabalhadas

- Uso do grid do Bootstrap  
- Escolha adequada de componentes  
- Responsividade  
- Organização visual da informação  
- Trabalho em equipe  
- Argumentação e tomada de decisão  
- Análise crítica de interface  

---

## Metodologia ativa utilizada

A atividade trabalha com:

- Aprendizagem baseada em problemas  
- Colaboração em pares  
- Prototipação  
- Avaliação entre colegas  
- Melhoria com base em feedback  

---

## Missão da dupla

Vocês receberam um problema real de interface.  
O desafio é criar uma solução usando Bootstrap.

A tela deve ser organizada, clara, responsiva e pensada para o usuário.  
O objetivo não é copiar um modelo pronto, mas propor uma interface funcional.

---

## Regras da atividade

A interface deve:

- Usar Bootstrap  
- Conter pelo menos 7 componentes do Bootstrap  
- Ser responsiva  
- Ter organização clara das informações  
- Priorizar a experiência do usuário  
- Ser apresentada ao final com explicação das escolhas  

---

## Entrega da dupla

A dupla deverá entregar:

- O arquivo da página  
- Um pequeno texto explicando as escolhas da interface  
- A versão ajustada após o feedback da outra dupla  

---

## Desafios

### Opção 1: Painel do professor

Crie uma interface para um professor visualizar:

- Avisos  
- Atividades pendentes  
- Materiais da aula  
- Botão para lançar presença  

### Opção 2: Agendamento de clínica

Crie uma tela simples para uma clínica organizar:

- Horários do dia  
- Nome do paciente  
- Status do atendimento  
- Botão de novo agendamento  

### Opção 3: Página de inscrição em evento

Crie uma tela para inscrição em um evento com:

- Informações do evento  
- Formulário  
- Botão de inscrição  
- Seção de dúvidas  

### Opção 4: Área do aluno (Matheus e João Hasegawa)

Crie uma página para o aluno visualizar:

- Próximas entregas 
- Notas recentes  
- Mensagens  
- Atalhos principais  

### Opção 5: Tela de suporte

Crie uma interface para registrar chamados com:

- Formulário  
- Prioridade  
- Lista de chamados  
- Status visual  

### Opção 6: Catálogo de produtos

Crie uma tela para exibir produtos com:

- Busca  
- Categorias  
- Cards  
- Botão de ação  

---

## Etapa 1 – Planejamento da solução (10/04/2026)

Antes de abrir o computador, a dupla deve responder:

- Quem é o usuário dessa tela?  
- Qual informação é mais importante?  
- O que precisa aparecer primeiro?  
- Quais componentes do Bootstrap ajudam melhor nesse caso?  
- Como a tela deve se comportar no celular?  

---

## Etapa 2 – Protótipo rápido (a definir)

A dupla deverá desenhar a estrutura da página em papel.

Eles devem marcar:

- Cabeçalho  
- Menu ou área de navegação  
- Conteúdo principal  
- Botões  
- Formulários ou cards  
- Organização em colunas  

Essa etapa ajuda a evitar que os alunos saiam programando sem pensar.

---

## Etapa 3 – Construção com Bootstrap (a definir)

Agora a dupla cria a tela.

### Exigências mínimas

Usar pelo menos:

- Sistema de grid  
- Botão  
- Card, tabela, alerta, formulário ou modal  
- Espaçamentos do Bootstrap  
- Algum recurso responsivo  

---

## Etapa 4 – Troca entre duplas (a definir)

Uma dupla testa a solução da outra e responde:

- O objetivo da tela ficou claro?  
- O que ficou mais fácil de entender?  
- O que ficou confuso?  
- O layout parece organizado?  
- A tela parece útil para o usuário?  
- O que poderia melhorar?  

---

## Etapa 5 – Ajuste final (a definir)

Depois de receber o feedback, a dupla melhora sua interface.

Aqui está uma das partes mais importantes da atividade, porque os alunos percebem que criar interface não é apenas “montar componentes”, mas melhorar a experiência de uso.

---

## Etapa 6 – Apresentação rápida (a definir)

Cada dupla apresenta em até 2 minutos:

- Qual era o problema  
- Como pensou na organização da tela  
- Quais componentes usou  
- O que mudou após o feedback
